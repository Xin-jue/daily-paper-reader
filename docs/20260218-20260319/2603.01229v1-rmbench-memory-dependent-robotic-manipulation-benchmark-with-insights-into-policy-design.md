## 摘要
近年来，机器人操纵策略取得了快速进展，但大多数现有方法对记忆能力的考虑有限。因此，它们难以解决需要对历史观测进行推理并随时间维持任务相关信息的任务，而这些是现实世界操纵场景中的常见需求。尽管已经提出了一些具备记忆意识的策略，但对记忆依赖型操纵的系统性评估仍有待探索，且架构设计选择与记忆性能之间的关系尚不明确。为了填补这一空白，我们推出了 RMBench，这是一个包含 9 个操纵任务的仿真基准，涵盖了多个记忆复杂度级别，能够对策略的记忆能力进行系统性评估。我们进一步提出了 Mem-0，这是一种具有显式记忆组件的模块化操纵策略，旨在支持受控的消融研究。通过广泛的仿真和真实世界实验，我们识别了现有策略中与记忆相关的局限性，并就架构设计选择如何影响记忆性能提供了经验性见解。网站见 https://rmbench.github.io/。

## Abstract
Robotic manipulation policies have made rapid progress in recent years, yet most existing approaches give limited consideration to memory capabilities. Consequently, they struggle to solve tasks that require reasoning over historical observations and maintaining task-relevant information over time, which are common requirements in real-world manipulation scenarios. Although several memory-aware policies have been proposed, systematic evaluation of memory-dependent manipulation remains underexplored, and the relationship between architectural design choices and memory performance is still not well understood. To address this gap, we introduce RMBench, a simulation benchmark comprising 9 manipulation tasks that span multiple levels of memory complexity, enabling systematic evaluation of policy memory capabilities. We further propose Mem-0, a modular manipulation policy with explicit memory components designed to support controlled ablation studies. Through extensive simulation and real-world experiments, we identify memory-related limitations in existing policies and provide empirical insights into how architectural design choices influence memory performance. The website is available at https://rmbench.github.io/.

当前的机器人操纵策略（如 Pi0.6, RDT2）在短时、精细操作上表现出色，但大多基于**马尔可夫假设（Markovian assumption）**，即决策仅依赖于当前或极短窗口内的观测。然而，现实任务往往是非马尔可夫的，需要机器人具备**长时记忆能力**（例如：记住之前放置物品的位置、在多次尝试失败后调整策略）。
*   **研究动机**：现有基准测试（如 LIBERO-Long）虽涉及长时程任务，但关键信息通常始终可见，不强制要求记忆；而专门的记忆基准（如 MemoryBench）存在难以复现或设计原则不明确的问题。
*   **核心目标**：系统性地评估机器人策略的记忆能力，并探索什么样的架构设计能有效提升记忆性能。

#### 2. 方法论：核心思想与关键技术
论文提出了一个“评价标准 + 任务基准 + 模块化策略”的完整框架：
*   **任务记忆复杂度 (Task Memory Complexity, TMC)**：定义为 $M(n)$，其中 $n$ 表示解决任务所需保留的关键历史观测数量。$M(0)$ 为无记忆任务，$M(1)$ 需保留一个历史片段，$M(n)$ 需保留多个非局部的时间依赖信息。
*   **RMBench 基准测试**：基于 RoboTwin 2.0 仿真平台，设计了 9 个双臂操纵任务，涵盖 $M(1)$（如：观察并抓取、交换方块）和 $M(n)$（如：电池尝试安装、按键计数、方块排序尝试）两个难度等级。
*   **Mem-0 策略架构**：采用双系统模块化设计：
    *   **规划模块 (Planning Module)**：使用 VLM（Qwen3-VL-8B）结合“关键记忆窗口（Key Memory Window）”，记录已完成子任务的描述和图像，进行高层逻辑推理。

    *   **子任务结束分类器 (Subtask End Classifier)**：一个轻量级 MLP，用于检测子任务是否完成，从而触发下一轮规划，实现闭环控制。

*   **仿真实验**：在 RMBench 的 9 个任务上进行。

#### 5. 实验数量与充分性
*   **实验规模**：涵盖了从简单到复杂的 9 个仿真任务和 3 个实物任务，每种方法在每个任务上进行 100 次 rollout，数据量足以支撑统计显著性。

#### 6. 主要结论与发现
*   **现有模型局限性**：传统策略在非马尔可夫任务上表现极差，因为它们无法处理超出固定窗口的历史信息。
*   **记忆组件的作用**：
    *   **锚点记忆**至关重要，移除它会导致策略在长时程任务中迷失目标。
    *   **滑动记忆**有助于捕捉运动趋势，但在某些对初始状态敏感的任务中可能引入干扰。
    *   **关键记忆**是 $M(n)$ 类任务成功的核心，决定了高层规划的正确性。
*   **性能提升**：Mem-0 在 $M(1)$ 任务上平均领先基准 38.4%，在 $M(n)$ 任务上领先 21.2%。

#### 7. 优点与亮点
*   **理论贡献**：首次提出了“任务记忆复杂度 (TMC)”这一量化指标，为机器人任务设计提供了科学指导。
*   **基准价值**：RMBench 填补了机器人领域缺乏系统性记忆评估工具的空白，且任务设计具有高度的复现性和挑战性。
*   **架构创新**：Mem-0 的双系统设计巧妙地结合了高层语义推理和底层精细动作，且模块化设计便于后续研究者进行组件替换和性能分析。

#### 8. 不足与局限
*   **语义理解瓶颈**：在需要极强语义辨析的任务（如“观察并抓取”中区分颜色形状极近的物体）中，Mem-0 仍逊色于大规模预训练模型。

*   **反馈机制**：目前主要依赖视觉，缺乏触觉或本体感受反馈，这在处理遮挡或微小接触任务时是短板。
