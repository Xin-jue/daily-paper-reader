## 摘要
视觉-语言-动作（VLA）模型是多模态机器人任务控制器，在给定指令和视觉输入的情况下，生成一系列低级控制动作（或电机命令），使机器人能够在物理环境中执行请求的任务。这些系统从多个角度面临测试预言机（test oracle）问题。一方面，必须为每个指令提示定义一个测试预言机，这是一种复杂且难以泛化的方法。另一方面，当前最先进的预言机通常捕获世界的符号表示（例如机器人和物体状态），从而能够评估任务的正确性，但无法评估其他关键方面，例如 VLA 驱动机器人执行任务的质量。在本文中，我们探讨了蜕变测试（MT）是否可以缓解这种情况下的测试预言机问题。为此，我们提出了两种蜕变关系模式和五种蜕变关系，以评估测试输入的更改是否会影响 VLA 驱动机器人的原始轨迹。一项涉及五个 VLA 模型、两个模拟机器人和四个机器人任务的实证研究表明，MT 可以通过自动检测各种类型的故障（包括但不限于未完成的任务）来有效缓解测试预言机问题。更重要的是，所提出的蜕变关系（MRs）具有通用性，使得该方法适用于不同的 VLA 模型、机器人和任务，即使在缺乏测试预言机的情况下也是如此。

## Abstract
Vision-Language-Action (VLA) models are multimodal robotic task controllers that, given an instruction and visual inputs, produce a sequence of low-level control actions (or motor commands) enabling a robot to execute the requested task in the physical environment. These systems face the test oracle problem from multiple perspectives. On the one hand, a test oracle must be defined for each instruction prompt, which is a complex and non-generalizable approach. On the other hand, current state-of-the-art oracles typically capture symbolic representations of the world (e.g., robot and object states), enabling the correctness evaluation of a task, but fail to assess other critical aspects, such as the quality with which VLA-enabled robots perform a task. In this paper, we explore whether Metamorphic Testing (MT) can alleviate the test oracle problem in this context. To do so, we propose two metamorphic relation patterns and five metamorphic relations to assess whether changes to the test inputs impact the original trajectory of the VLA-enabled robots. An empirical study involving five VLA models, two simulated robots, and four robotic tasks shows that MT can effectively alleviate the test oracle problem by automatically detecting diverse types of failures, including, but not limited to, uncompleted tasks. More importantly, the proposed MRs are generalizable, making the proposed approach applicable across different VLA models, robots, and tasks, even in the absence of test oracles.

### 1. 核心问题与整体含义（研究动机和背景）
论文聚焦于**视觉-语言-动作（VLA）模型**在机器人控制领域的测试难题。VLA模型能将自然语言指令和视觉图像直接转化为机器人的底层控制动作，但其评估面临严重的**测试预言机（Test Oracle）问题**：
*   **难以泛化**：现有的符号预言机（Symbolic Oracles）通常针对特定任务编写（如检查物体是否被提起），难以应对人类指令的多样性和模糊性。
*   **质量评估缺失**：符号预言机仅关注任务是否最终完成（二进制结果），无法评估执行过程中的轨迹质量、平稳性、安全性及对环境扰动的鲁棒性。
*   **研究动机**：引入**蜕变测试（Metamorphic Testing, MT）**，通过定义输入与输出之间的逻辑关系（蜕变关系），在无需知道确切正确输出的情况下检测系统故障。

### 2. 方法论
核心思想是利用机器人运动轨迹的**一致性**和**变异性**来识别故障。论文提出了两种蜕变关系模式（MRP）及五种具体的蜕变关系（MR）：

*   **MRP1：轨迹一致性模式（Trajectory Consistency, TC）**
    *   **核心思想**：对输入进行不改变语义的修改，机器人的执行轨迹应保持基本不变。
    *   **MR1（同义词替换）**：更换指令中的动词（如“Pick”改为“Grab”），轨迹应一致。
    *   **MR2（非干扰物体添加）**：在远离目标处增加无关物体，轨迹应一致。
    *   **MR3（光照亮度变化）**：改变场景亮度，轨迹应一致。
*   **MRP2：轨迹变异性模式（Trajectory Variation, TV）**
    *   **核心思想**：对输入进行特定修改，轨迹应产生可预测的变化。
    *   **MR4（任务否定/反转）**：指令改为“不要执行某动作”，机器人应停止或大幅改变轨迹。
    *   **MR5（目标物体移位）**：移动目标物体位置，机器人轨迹应随之成比例调整。
*   **关键技术细节**：
    *   使用 **Fréchet 距离** 来衡量原始轨迹与后续轨迹之间的差异。
    *   引入容差阈值 $\delta$（严格、中等、宽松三档）来判定是否违反蜕变关系。

### 3. 实验设计
*   **受试模型（VLA Models）**：选取了5个最先进的模型：OpenVLA, $\pi_0$, SpatialVLA, GR00T-N1.5, EO-1。
*   **机器人平台**：Google Robot (Everyday Robot) 和 WidowX 机械臂。
*   **实验场景与任务**：基于 **SimplerEnv** 仿真环境，涵盖四类任务：拿起（Pick up）、移近（Move Near）、放置在上方（Put on）、放入内部（Put in）。
*   **对比基准（Benchmark）**：将 MT 的检测结果与传统的**符号预言机**进行对比，分析两者在故障检测种类上的重合度与互补性。

### 4. 资源与算力
论文详细记录了实验的硬件配置与耗时：
*   **硬件1**：AMD EPYC 7773X CPU + **NVIDIA RTX A6000 GPU (48 GB)**，用于运行 $\pi_0$ 和 OpenVLA。
*   **硬件2**：AMD Ryzen 9 7900X CPU + **NVIDIA GeForce RTX 4080 SUPER (16 GB)**，用于运行 GR00T-N1.5 和 SpatialVLA。
*   **总算力消耗**：总计执行时间为 219,168 秒，约合 **61 小时 GPU 运行时间**。
*   **单次测试耗时**：不同模型差异较大，从 7.5 秒（GR00T）到 35 秒（SpatialVLA）不等。

### 5. 实验数量与充分性
*   **实验规模**：共选取了 1,864 个原始测试用例（均为符号预言机判定成功的案例），并生成了 **9,320 个后续测试用例对**。
*   **充分性**：实验覆盖了多种主流 VLA 架构、两种不同的机器人形态以及四种代表性操纵任务。
*   **客观性**：通过固定随机种子确保环境初始化一致，并采用 Cochran 抽样法抽取 192 个故障案例进行人工标注，以构建故障分类法，保证了定性分析的统计学意义。

### 6. 主要结论与发现
*   **MT 的有效性**：MT 能有效检测出符号预言机漏掉的故障。在“中等”阈值下，MT 额外发现了大量行为不一致的案例。
*   **互补性**：符号预言机擅长检测“任务是否完成”，而 MT 擅长检测“执行质量”（如轨迹抖动、不必要的绕路、对无关物体的过度反应）。
*   **MR 表现差异**：环境扰动（MR2）和语义否定（MR4）在暴露故障方面最有效。
*   **故障分类**：通过 MT 发现了三类主要失效：**操纵失效**（抓取不稳）、**运动失效**（轨迹次优、控制不稳定、碰撞）和**规划推理失效**（指令理解错误）。

### 7. 优点（亮点）
*   **任务无关性**：提出的 MRP 模式具有通用性，不需要为每个新任务重新编写复杂的预言机。
*   **关注过程质量**：填补了机器人测试中“轨迹质量评估”的空白，能够识别出虽然任务成功但过程危险或低效的行为。
*   **系统性分类**：建立了一套针对 VLA 模型的故障分类法（Taxonomy），为后续研究提供了标准。

### 8. 不足与局限
*   **阈值敏感性**：MT 的效果高度依赖于 Fréchet 距离阈值 $\delta$ 的设定，手动调优可能存在主观性。
*   **仿真与现实的差距（Sim-to-Real）**：实验完全在仿真环境中进行，未在真实物理机器人上验证环境噪声对 MR 的影响。
*   **非确定性挑战**：VLA 模型（尤其是基于扩散模型的）具有内在随机性，如何区分“正常的行为波动”与“蜕变关系违背”仍是一个挑战。
*   **计算开销**：由于需要多次运行（原始+后续），测试成本比单次运行的符号预言机更高。
