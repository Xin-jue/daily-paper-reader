## 摘要
通过明确且可执行的里程碑来衡量任务进度，对于鲁棒的机器人操作至关重要。这种进度感知能力使模型能够确定当前任务状态，预测可验证的中间状态，并在进度停滞时检测并从故障中恢复。为了实现这一能力，我们引入了 See, Plan, Rewind (SPR)，这是一个进度感知的视觉-语言-动作框架，它能动态地将语言指令转化为一系列空间子目标。SPR 通过一个持续的核心循环运行：观察（Seeing）当前状态和即将到来的里程碑，规划（Planning）通往下一个 2D 路径点的轨迹，并在失败时通过对照预期序列监测进度，回溯（Rewinding）到可恢复状态。这种闭环方法实现了鲁棒的错误纠正，且无需额外的训练数据或辅助模型。大量实验证明了该框架的有效性、泛化性和鲁棒性：在 LIBERO 基准测试中，SPR 的表现比 MolmoAct 基准高出 5%。在具有未见指令和初始状态的挑战性 LIBERO-Plus 基准测试中，SPR 以最小的性能下降实现了最先进的鲁棒性，超越了 OpenVLA-OFT 和 UniVLA，展示了卓越的分布外鲁棒性。

## Abstract
Measurement of task progress through explicit, actionable milestones is critical for robust robotic manipulation. This progress awareness enables a model to ground its current task status, anticipate verifiable intermediate states, and detect and recover from failures when progress stalls. To embody this capability, we introduce See, Plan, Rewind (SPR), a progress-aware vision-language-action framework that dynamically grounds language instructions into a sequence of spatial subgoals. SPR operates through a continuous core cycle, Seeing the current state and upcoming milestone, Planning a trajectory towards the next 2D waypoint, and Rewinding to a recoverable state upon failure by monitoring progress against the expected sequence. This closed-loop approach enables robust error correction without requiring additional training data or auxiliary models. Extensive experiments demonstrate the framework's effectiveness, generalization and robustness: SPR outperforms the MolmoAct baseline by 5\% on the LIBERO benchmark. On the challenging LIBERO-Plus benchmark with unseen instructions and initial states, SPR achieves state-of-the-art robustness with the smallest performance drop, surpassing OpenVLA-OFT and UniVLA, demonstrating superior out-of-distribution robustness.

### 1. 核心问题与整体含义（研究动机和背景）
机器人操作通常涉及复杂的长程任务，现有的视觉-语言-动作（VLA）模型虽然能执行基本任务，但往往缺乏对任务进度的**显式、定量感知**。当机器人遇到碰撞、抓取失败或环境变化时，由于无法准确判断当前处于任务的哪个阶段，容易陷入无效的重复动作或彻底失败。
**核心动机**：赋予机器人“进度感知”能力，使其能够将抽象指令分解为可验证的空间里程碑（子目标），并在进度停滞时自主“回溯”到可恢复状态，从而提升操作的鲁棒性和分布外（OOD）泛化能力。

### 2. 方法论：核心思想与技术细节
论文提出了 **SPR (See, Plan, Rewind)** 框架，其核心是一个闭环执行循环：

*   **See（观察）**：模型接收图像和指令，输出深度感知、剩余子任务数量、每个子任务的语义描述及其在图像中的 2D 坐标。这为任务提供了空间锚点。
*   **Plan（规划）**：不同于直接规划到终点，SPR 规划通往**下一个子目标**的 2D 轨迹（包含 1-5 个路径点），这降低了长程任务的规划难度。
*   **Rewind（回溯）**：
    *   **异常检测**：通过状态记录器监测预测的子任务数（若增加则视为失败）和 2D 轨迹（若长时间不变则视为停滞）。
    *   **恢复策略**：一旦检测到异常，模型切换到“回溯”模式，执行 $N=3$ 步“返回初始位置”的动作，使机器人脱离错误状态并重新尝试。
*   **自动化数据流水线**：
    *   利用夹具状态（开/合）或多模态大模型（如 Gemini-3）自动分割演示视频。
    *   结合 DINOv3 和 SAM 提取夹具的 2D 空间坐标，无需人工标注即可生成带有空间子目标的训练数据。

### 6. 主要结论与发现
*   **性能提升**：在 LIBERO 基准上，SPR 比 MolmoAct 平均高出 5%，在复杂的 Long 任务中提升尤为显著（+8.2%）。

### 7. 优点（亮点）
*   **无需额外数据**：回溯机制通过反转成功演示数据生成，不需要专门收集失败恢复数据。
*   **空间与语义结合**：将抽象的语言指令转化为具体的 2D 坐标，使进度监测变得可量化、可解释。
*   **闭环纠错**：引入了类似人类“退一步海阔天空”的思维，通过简单的回溯动作有效解决了分布外状态导致的死循环问题。

### 8. 不足与局限
*   **精度限制**：由于使用离散动作 Token，在需要毫米级精度的任务（如精准对齐）中仍存在局限。
*   **物理卡死无法解决**：如果机器人被物理障碍物彻底卡死（如夹具卡在缝隙中），单纯的“回溯”指令可能因无法产生位移而失效。
*   **动作与规划不一致**：在某些失败案例中，模型虽然预测了正确的 2D 路径点，但生成的底层动作却偏离了该路径（Hallucination 现象）。
