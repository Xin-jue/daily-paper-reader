## 摘要
我们提出了 TiPToP，这是一个可扩展的模块化系统，它将预训练的视觉基础模型与现有的任务与运动规划器（TAMP）相结合，直接根据输入的 RGB 图像和自然语言指令解决多步操作任务。我们的系统旨在简单易用：它可以在不到一小时的时间内安装并在标准的 DROID 平台上运行，并能以极小的代价适配到新的机器人本体。我们在仿真和真实世界的 28 个桌面操作任务中评估了 TiPToP（该系统不需要任何机器人数据），发现其表现与 π_{0.5}-DROID 相当甚至更优，后者是一个在 350 小时特定本体演示数据上进行微调的视觉-语言-动作（VLA）模型。TiPToP 的模块化架构使我们能够在组件级别分析系统的故障模式。我们分析了 173 次实验评估的结果，并确定了改进方向。我们开源了 TiPToP，以进一步推动模块化操作系统以及学习与规划之间更紧密集成的研究。项目网站和代码：https://tiptop-robot.github.io

## Abstract
We present TiPToP, an extensible modular system that combines pretrained vision foundation models with an existing Task and Motion Planner (TAMP) to solve multi-step manipulation tasks directly from input RGB images and natural-language instructions. Our system aims to be simple and easy-to-use: it can be installed and run on a standard DROID setup in under one hour and adapted to new embodiments with minimal effort. We evaluate TiPToP -- which requires zero robot data -- over 28 tabletop manipulation tasks in simulation and the real world and find it matches or outperforms $π_{0.5}\text{-DROID}$, a vision-language-action (VLA) model fine-tuned on 350 hours of embodiment-specific demonstrations. TiPToP's modular architecture enables us to analyze the system's failure modes at the component level. We analyze results from an evaluation of 173 trials and identify directions for improvement. We release TiPToP open-source to further research on modular manipulation systems and tighter integration between learning and planning. Project website and code: https://tiptop-robot.github.io

长期以来，机器人领域的目标是构建一个“开箱即用”的系统：无需针对特定物体、环境或机器人本体进行微调，即可执行自然语言指令。
*   **现有技术的局限：** 视觉-语言-动作模型（VLA，如 $\pi_0$）虽然具备通用性，但需要海量的机器人演示数据，且难以诊断失败原因；传统的任务与运动规划（TAMP）虽然逻辑严密，但通常依赖预先建模的几何信息，缺乏对未知场景的泛化能力。
*   **研究动机：** 结合预训练视觉基础模型（感知）与 GPU 加速的 TAMP（规划），创建一个无需任何机器人训练数据、仅靠 RGB 图像即可完成复杂多步任务的模块化系统。

#### 2. 论文提出的方法论
TiPToP 系统由三个核心模块组成，输入为双目 RGB 图像和自然语言指令，输出为关节轨迹：
*   **感知模块 (Perception)：**
    *   **3D 视觉分支：** 使用 *FoundationStereo* 进行深度估计，生成 3D 点云；使用 *M2T2* 预测 6-DoF 抓取位姿。
    *   **语义分支：** 使用 *Gemini 1.5 Pro* 进行目标检测和目标接地（Goal Grounding），将指令转化为逻辑谓词（如 `On(A, B)`）；使用 *SAM-2* 生成精确的分割掩码。
    *   **融合：** 通过凸包（Convex Hull）重建物体网格，并将抓取位姿分配给特定物体。
*   **规划模块 (Planning)：**
    *   使用 **cuTAMP**（GPU 并行化的 TAMP 算法）在离散动作序列（骨架）和连续几何约束（抓取、放置、避障）之间进行联合优化。
    *   利用 **cuRobo** 生成无碰撞的时间参数化轨迹。
*   **执行模块 (Execution)：**
    *   采用自定义的关节阻抗控制器，精确跟踪规划好的轨迹。系统目前采用**开环执行**模式。

#### 6. 论文的主要结论与发现
*   **性能超越：** 在语义推理和多步规划任务中，TiPToP 的成功率显著优于经过大量微调的 $\pi_{0.5}$-DROID。
*   **执行效率：** TiPToP 生成的轨迹通常比 VLA 模型的反应式控制更直接，完成任务的时间更短。
*   **互补性：** 发现 TiPToP 擅长几何推理和长程规划，而 VLA 在处理抓取滑动等需要实时反馈的场景中表现更好。

#### 7. 优点
*   **模块化与可解释性：** 失败可以追溯到具体的感知或规划组件，便于针对性调试。
*   **零样本泛化：** 无需昂贵的机器人数据采集，直接利用视觉大模型的常识和推理能力。
*   **高性能规划：** 借助 GPU 加速，解决了传统 TAMP 计算缓慢的问题。
*   **开源贡献：** 提供了完整的开源工具链，降低了规划类机器人研究的门槛。

#### 8. 不足与局限
*   **开环限制：** 系统在执行过程中不进行视觉反馈（Open-loop），一旦物体发生意外移动或抓取打滑，系统无法自动纠正。
*   **几何近似误差：** 使用凸包近似物体形状会导致在处理凹面物体（如碗、香蕉）时出现碰撞检测错误。
*   **单视角依赖：** 仅依靠初始时刻的单视角图像，容易受到遮挡影响，且点云质量受限于相机视角。
*   **动态环境适应性：** 无法处理在执行过程中发生变化的环境。
