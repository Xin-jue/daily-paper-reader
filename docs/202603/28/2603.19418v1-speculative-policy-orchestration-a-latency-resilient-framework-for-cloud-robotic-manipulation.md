## 摘要
云机器人技术使机器人能够将高维运动规划和推理卸载到远程服务器。然而，对于需要高频控制的连续操纵任务，网络延迟和抖动会严重破坏系统的稳定性，导致指令匮乏和不安全的物理执行。为解决这一问题，我们提出了投机性策略编排（SPO），这是一种抗延迟的云边框架。SPO 利用云端托管的世界模型预计算未来的运动学路点，并将其流式传输到本地边缘缓冲区，从而将执行频率与网络往返时间解耦。为了减轻预测漂移引起的不安全执行，边缘节点采用了一个 ε-管验证器（ε-tube verifier），严格限制运动学执行误差。该框架结合了自适应时界缩放机制，根据实时跟踪误差动态扩展或缩小投机性预取深度。我们在模拟网络延迟下，通过连续 RLBench 操纵任务对 SPO 进行了评估。结果表明，即使部署精度适中的学习模型，与阻塞式远程推理相比，SPO 也能将网络引起的空闲时间减少 60% 以上。此外，与静态缓存基准相比，SPO 丢弃的云端预测减少了约 60%。最终，SPO 在保持有界物理安全性的同时，实现了流畅、实时的云机器人控制。

## Abstract
Cloud robotics enables robots to offload high-dimensional motion planning and reasoning to remote servers. However, for continuous manipulation tasks requiring high-frequency control, network latency and jitter can severely destabilize the system, causing command starvation and unsafe physical execution.   To address this, we propose Speculative Policy Orchestration (SPO), a latency-resilient cloud-edge framework. SPO utilizes a cloud-hosted world model to pre-compute and stream future kinematic waypoints to a local edge buffer, decoupling execution frequency from network round-trip time. To mitigate unsafe execution caused by predictive drift, the edge node employs an $ε$-tube verifier that strictly bounds kinematic execution errors. The framework is coupled with an Adaptive Horizon Scaling mechanism that dynamically expands or shrinks the speculative pre-fetch depth based on real-time tracking error.   We evaluate SPO on continuous RLBench manipulation tasks under emulated network delays. Results show that even when deployed with learned models of modest accuracy, SPO reduces network-induced idle time by over 60% compared to blocking remote inference. Furthermore, SPO discards approximately 60% fewer cloud predictions than static caching baselines. Ultimately, SPO enables fluid, real-time cloud-robotic control while maintaining bounded physical safety.

*   **研究背景**：随着深度神经网络（如扩散策略、Transformer）在机器人操纵中的应用，计算需求剧增，促使研究者将策略推理卸载到云端。

*   **整体含义**：本文旨在建立一种“抗延迟”的云边协同框架，使机器人能在不稳定的网络环境下，依然保持流畅且安全的连续动作执行。

### 2. 方法论：核心思想与关键技术
SPO 框架借鉴了计算机体系结构中的“投机执行”和大模型中的“投机采样”概念，核心思想是**“云端预测未来，边缘验证执行”**。
*   **云端投机生成（Speculative Generation）**：云端不仅计算当前动作，还利用一个**世界模型（World Model）**自回归地预测未来 $K$ 个步长的状态和动作序列（即投机块），并流式传输至边缘。
*   **边缘 $\epsilon$-管验证器（$\epsilon$-tube Verifier）**：边缘节点维护一个缓冲区。在每个控制周期，验证器对比机器人当前真实状态 $s_t$ 与缓存中的预测状态 $\hat{s}_t$。若误差在预设的 $\epsilon$ 范围内（即处于“安全管”内），则执行缓存动作；否则判定为“投机失败”，清空缓存并触发安全停止。
*   **自适应视界缩放（AHS）**：引入类似 TCP 拥塞控制的“加性增、乘性减”逻辑。当跟踪误差小时，逐步增加投机深度 $K$ 以掩盖延迟；当发生验证失败（接触物体或扰动）时，根据误差严重程度大幅缩减 $K$，以降低开销并确保安全。

### 3. 实验设计
*   **实验场景**：基于 **RLBench** 机器人学习基准环境，选择了三个复杂度递增的任务：

    2.  **Insert Onto Square Peg**（套圈）：中等复杂度，涉及精密对准。
    3.  **Put All Groceries In Cupboard**（收纳杂物）：高复杂度，多物体、多阶段操纵。
*   **网络环境**：模拟广域网（WAN），设置 **150ms 基础延迟**及 **±30ms 随机抖动**。
*   **对比基准（Baselines）**：
    *   **Blocking**：传统的阻塞式远程推理（每步都等云端返回）。
    *   **T1-SC**：仅预取 1 步的简单投机。
    *   **NFTC**：静态预取固定长度（$K=10$）的朴素缓存策略。

*   **模型架构**：学习型模型采用了轻量级的 **3 层 MLP**（隐藏层维度 512），用于自回归预测状态和动作。

### 5. 实验数量与充分性
*   **实验规模**：针对三个任务分别进行了多轮测试，涵盖了成功率、空闲时间、缓存命中率、跟踪误差等多个维度的定量分析。
*   **充分性评价**：实验设计较为充分。作者不仅对比了不同任务复杂度，还对比了理想模型与存在预测漂移的学习模型。通过消融实验展示了 AHS 机制在平衡“延迟掩盖”与“计算浪费”之间的作用。实验过程考虑了网络随机抖动，具有较好的客观性和现实参考价值。

### 6. 主要结论与发现
*   **显著降低空闲时间**：相比阻塞式推理，SPO 将网络诱导的机器人空闲时间减少了 **60% 以上**。
*   **提升资源效率**：相比静态缓存（NFTC），SPO 通过自适应缩放机制减少了约 **60% 的无效云端预测**（即被丢弃的错误预测）。
*   **安全性保障**：$\epsilon$-管验证器能有效识别物理接触导致的预测偏差，防止机器人在开环状态下发生危险碰撞。
*   **鲁棒性**：即使在世界模型精度有限的情况下，SPO 依然能通过频繁的边缘验证维持系统的稳定性。

### 7. 优点
*   **解耦控制与通信**：成功将高频物理执行频率与低频/高延迟的网络往返时间解耦。
*   **确定性安全边界**：不依赖神经网络输出的概率置信度（通常不准），而是采用基于物理状态的硬性几何约束（$\epsilon$-tube）。
*   **动态自适应**：AHS 机制能根据任务难度自动调整“激进”程度，在自由空间运动时深预取，在接触作业时谨慎执行。

### 8. 不足与局限
*   **仿真与现实差距**：主要实验在 RLBench 仿真中完成，虽然模拟了网络延迟，但尚未在真实物理机器人上验证传感器噪声和复杂的 sim-to-real 动力学差异。
*   **安全停止策略**：算法中采用的“零速度保持（SafeStop）”在真实硬件上可能导致机械应力过大，实际部署需配合阻抗控制等平滑减速手段。
*   **模型依赖性**：虽然框架本身模型无关，但其性能高度依赖于“世界模型”的预测精度。如果世界模型极差，系统会频繁退化为阻塞模式。
