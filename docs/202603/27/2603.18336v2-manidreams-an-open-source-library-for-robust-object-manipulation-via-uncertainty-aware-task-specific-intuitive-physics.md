## 摘要
动力学模型，无论是模拟器还是学习到的世界模型，长期以来一直是机器人操控的核心，但大多数模型侧重于最小化预测误差，而非应对一个更根本的挑战：现实世界的操控本质上具有不确定性。我们认为，不确定性下的鲁棒操控从根本上是一个集成问题：不确定性必须在规划循环中进行表示、传播和约束，而不仅仅是在训练过程中被抑制。我们提出并开源了 ManiDreams，这是一个基于直觉物理模型进行不确定性感知操控规划的模块化框架。它通过可组合的抽象实现了这种集成，包括分布状态表示、后端无关的动力学预测以及用于动作优化的声明式约束规范。该框架明确解决了三种不确定性来源：感知、参数和结构。它通过“采样-预测-约束”循环封装任何基础策略，根据分布结果评估候选动作，从而在无需重新训练的情况下增加鲁棒性。在 ManiSkill 任务上的实验表明，在各种扰动下，当强化学习（RL）基准性能显著下降时，ManiDreams 仍能保持鲁棒的表现。在推、取、接以及现实世界部署中的可运行示例证明了其在不同策略、优化器、物理后端和执行器之间的灵活性。该框架已在 https://github.com/Rice-RobotPI-Lab/ManiDreams 公开。

## Abstract
Dynamics models, whether simulators or learned world models, have long been central to robotic manipulation, but most focus on minimizing prediction error rather than confronting a more fundamental challenge: real-world manipulation is inherently uncertain. We argue that robust manipulation under uncertainty is fundamentally an integration problem: uncertainties must be represented, propagated, and constrained within the planning loop, not merely suppressed during training. We present and open-source ManiDreams, a modular framework for uncertainty-aware manipulation planning over intuitive physics models. It realizes this integration through composable abstractions for distributional state representation, backend-agnostic dynamics prediction, and declarative constraint specification for action optimization. The framework explicitly addresses three sources of uncertainty: perceptual, parametric, and structural. It wraps any base policy with a sample-predict-constrain loop that evaluates candidate actions against distributional outcomes, adding robustness without retraining. Experiments on ManiSkill tasks show that ManiDreams maintains robust performance under various perturbations where the RL baseline degrades significantly. Runnable examples on pushing, picking, catching, and real-world deployment demonstrate flexibility across different policies, optimizers, physics backends, and executors. The framework is publicly available at https://github.com/Rice-RobotPI-Lab/ManiDreams

*   **参数不确定性**（物理参数估计不准）。
*   **结构不确定性**（动力学模型不完美）。

#### 2. 方法论：核心思想与关键技术
ManiDreams 实现了“**采样-预测-约束**”（Sample-Predict-Constrain）的范式，其核心由四个可组合的抽象层组成：
*   **DRIS（领域随机化实例集）**：一种分布式的状态表示。它不使用单一的状态估计，而是维护一组具有不同物理参数和位姿假设的物体实例，以捕捉感知和参数不确定性。
*   **TSIP（任务特定直觉物理）**：后端无关的预测接口。它可以调用物理模拟器（如 ManiSkill3）或学习到的模型（如扩散模型）来预测 DRIS 在候选动作下的演化。
*   **笼式约束（Caging Constraints）**：用于界定不确定性的边界。它评估预测的分布是否满足任务安全要求（如物体是否保持在抓取范围内），提供连续代价和二值验证。
*   **求解器（Solver）**：结合采样器（如 PPO 策略、高斯分布）和优化器（如 MPPI、N-best 选择），在预测的分布结果中筛选出最优且满足约束的动作。

**算法流程**：
1.  **同步**：将执行器的当前状态同步到多个并行评估环境中。
2.  **采样**：从基础策略或采样器中生成 $N$ 个候选动作。
3.  **传播**：通过 TSIP 预测每个动作后 DRIS 的分布状态。
4.  **评估与约束**：利用笼式约束计算每个预测分布的代价和合法性。
5.  **执行**：选择最优动作并发送至执行器（模拟器或真实机器人）。

#### 6. 主要结论与发现
*   **鲁棒性显著提升**：在存在中等程度的感知噪声、延迟或物理参数偏差时，ManiDreams 的成功率远高于纯 PPO 基准。
*   **零样本迁移**：通过更换执行器（Executor），框架能够实现从仿真到真实机器人的零样本（Zero-shot）迁移，无需针对真实环境重新训练策略。
*   **模块化优势**：证明了 TSIP 后端（模拟器 vs 学习模型）和约束类型的可替换性，且不影响整体规划逻辑。

#### 7. 优点
*   **插件式架构**：高度模块化，用户可以轻松更换物理后端、策略或约束条件。
*   **不确定性显式化**：将不确定性作为“一等公民”处理，通过分布传播而非点估计来提高安全性。
*   **无需重训**：可以作为现有策略（如预训练的 RL 策略）的“包装层”，在不改变原始权重的情况下增强其鲁棒性。
*   **开源友好**：提供了详细的文档和多种任务示例。

#### 8. 不足与局限
*   **参数调节**：DRIS 的分布范围和笼式约束的参数仍需针对具体任务进行手动调节，缺乏自动配置机制。
*   **计算开销**：虽然在笔记本 GPU 上可运行，但对于高动态任务（如高速碰撞），当前的推理延迟（约 50-100ms）可能仍然过高。
*   **动力学连续性假设**：对于具有高度不连续动力学的任务，简单的分布采样可能无法完全捕捉复杂的物理演化。
