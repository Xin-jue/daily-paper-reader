## 摘要
部署在智能体环境中的大语言模型（LLMs）必须在不同的任务实例中运用多种能力。在这里，“能力”被定义为在轨迹中执行一个或多个动作，这些动作对于成功解决环境中的子集任务至关重要。许多现有方法要么依赖于并非针对模型在目标环境中实际能力缺陷的合成训练数据，要么直接在目标环境上进行训练，导致模型需要跨任务隐式地学习这些能力。我们提出了 TRACE（Turning Recurrent Agent failures into Capability-targeted training Environments，将智能体循环失败转化为针对能力的训练环境），这是一个用于特定环境智能体自我提升的端到端系统。TRACE 通过对比成功和失败的轨迹来自动识别缺失的能力，并为每种能力合成一个针对性的训练环境，该环境会根据是否运用了该能力提供奖励。随后，TRACE 在每个合成环境上通过强化学习（RL）训练 LoRA 适配器，并在推理时路由至相关的适配器。实验结果表明，TRACE 在不同环境中具有良好的泛化性，在 $τ^2$-bench（客户服务）上比基础智能体提升了 14.1 分，在 ToolSandbox（工具使用）上增加了 7 个满分，分别比最强基准模型高出 7.4 分 and 4 个满分。在相同数量的采样（rollouts）下，TRACE 的扩展效率优于基准模型，在 $τ^2$-bench 上分别比 GRPO 和 GEPA 高出 9.2 分和 7.4 分。

## Abstract
Large Language Models (LLMs) deployed in agentic environments must exercise multiple capabilities across different task instances, where a capability is performing one or more actions in a trajectory that are necessary for successfully solving a subset of tasks in the environment. Many existing approaches either rely on synthetic training data that is not targeted to the model's actual capability deficits in the target environment or train directly on the target environment, where the model needs to implicitly learn the capabilities across tasks. We introduce TRACE (Turning Recurrent Agent failures into Capability-targeted training Environments), an end-to-end system for environment-specific agent self-improvement. TRACE contrasts successful and failed trajectories to automatically identify lacking capabilities, synthesizes a targeted training environment for each that rewards whether the capability was exercised, and trains a LoRA adapter via RL on each synthetic environment, routing to the relevant adapter at inference. Empirically, TRACE generalizes across different environments, improving over the base agent by +14.1 points on $τ^2$-bench (customer service) and +7 perfect scores on ToolSandbox (tool use), outperforming the strongest baseline by +7.4 points and +4 perfect scores, respectively. Given the same number of rollouts, TRACE scales more efficiently than baselines, outperforming GRPO and GEPA by +9.2 and +7.4 points on $τ^2$-bench.

### 1. 核心问题与整体含义
*   **研究动机**：尽管大语言模型（LLM）在智能体任务中表现出色，但它们往往在特定环境下存在“能力短板”（如无法正确解析复杂的 JSON、无法处理权限错误等）。
*   **核心挑战**：现有的提升方法（如直接在目标环境进行强化学习或使用通用合成数据）效率低下。原因在于：1. 目标环境的奖励信号稀疏，难以归因到具体能力；2. 通用合成数据并非针对模型实际的弱点。
*   **核心目标**：开发一个端到端的系统，能够自动诊断模型在特定环境下的能力缺陷，并针对性地生成训练环境来“补课”。

### 2. 方法论
TRACE（Turning Recurrent Agent failures into Capability-targeted training Environments）包含四个关键步骤：
*   **对比能力识别（Contrastive Identification）**：
    *   系统对比模型在目标环境中的成功轨迹（$D^+$）和失败轨迹（$D^-$）。
    *   利用 LLM 分析智能体在失败案例中缺失的候选能力，并计算“对比差距”（Contrastive Gap）：$b\Delta(c) = cER^-(c) - cER^+(c)$，即该能力在失败组中的缺失率与成功组中的缺失率之差。
    *   优先训练那些对比差距大且覆盖失败案例广（Coverage）的高影响能力。
*   **合成针对性训练环境（Environment Synthesis）**：
    *   为每个识别出的能力缺陷自动生成一个独立的合成环境。
    *   **关键技术**：环境保留了目标环境的工具 Schema 和交互协议，但通过程序化生成（Procedural Generation）确保任务成功高度依赖于该特定能力。
    *   **奖励设计**：设计密集的、可验证的奖励信号（如数据库状态变化、工具参数准确性）。
*   **强化学习训练（RL Training）**：
    *   针对每个能力，在对应的合成环境中使用 **GRPO**（Group Relative Policy Optimization）算法训练一个独立的 **LoRA 适配器**。
*   **能力组合与路由（Routing）**：
    *   在推理阶段，不合并模型权重，而是使用冻结的基础模型作为“路由器”，根据当前任务描述动态选择并激活最相关的 LoRA 适配器。

### 3. 实验设计
*   **数据集/场景**：
    *   **$\tau^2$-Bench**：模拟真实世界的客户服务工作流（航空和零售领域），涉及复杂的工具调用和策略约束。
    *   **ToolSandBox**：评估智能体在状态化环境中的工具使用能力（如处理权限错误、日期时间推理）。
*   **对比方法**：
    *   **Base Model**：原始 Qwen3-30B 模型。
    *   **Direct GRPO**：直接在目标基准环境上进行强化学习。
    *   **ADP (Agent Data Protocol)**：在多样化的公开智能体轨迹上进行 SFT。
    *   **AWM (Agent World Model)**：在与基准无关的通用合成环境中进行 RL。
    *   **GEPA**：一种先进的提示词进化/优化方法。
    *   **多种合并策略**：如 CORE-TSV（模型合并）、多能力混合训练等。

### 4. 资源与算力
*   **基础模型**：Qwen3-30B-A3B-Instruct-2507。
*   **硬件资源**：使用了 **4 到 8 张 NVIDIA A100-80GB GPU**。
*   **训练细节**：每个能力适配器最多训练 40 个迭代。使用了 AdamW 优化器（学习率 $10^{-5}$），并开启了梯度检查点（Gradient Checkpointing）和分布式数据并行。
*   **推理开销**：路由过程仅增加几秒钟的延迟，在多轮对话中开销可忽略不计。

### 5. 实验数量与充分性
*   **实验规模**：在两个主流智能体基准上进行了全面测试，涵盖了 164 个 $\tau^2$-Bench 任务和 129 个 ToolSandBox 场景。
*   **稳定性验证**：为了验证能力识别的稳定性，作者进行了 **10 次独立运行** 的对比分析，证明系统能稳定识别出相同的核心能力缺陷（如结构化数据推理、多步任务完成等）。
*   **消融实验**：对比了“路由”与“模型合并/蒸馏”的效果，证明了路由策略在保持各能力独立性方面的优越性。
*   **扩展性分析**：展示了性能随采样数量（Rollouts）和能力数量增加的扩展曲线。实验设计客观、公平，且对比了当前最前沿的合成数据和提示词优化方法。

### 6. 主要结论与发现
*   **性能飞跃**：TRACE 在 $\tau^2$-Bench 上比基础模型提升了 **14.1%** 的成功率，在 ToolSandBox 上增加了 **7 个** 满分案例，显著优于所有基准方法。
*   **训练效率极高**：在相同采样数量下，TRACE 的表现远超直接 RL（GRPO）和提示词优化（GEPA）。例如，仅用 2-4 个 LoRA 适配器（每个仅更新 5.3% 的参数）即可获得巨大收益。
*   **针对性胜过通用性**：实验证明，针对模型特定短板训练“微型环境”，比在大规模通用合成数据上训练更有效。
*   **路由优于合并**：尝试将多个能力合并到一个模型中往往会导致性能下降，而动态路由能更好地保留专业化能力。

### 7. 优点
*   **闭环自改进**：实现了从“发现问题”到“生成环境”再到“解决问题”的完全自动化。
*   **高数据效率**：通过隔离能力，将复杂的任务分解为易于学习的子模块，极大地缓解了强化学习中的信用分配（Credit Assignment）问题。
*   **模块化与可扩展性**：LoRA 适配器架构允许在不干扰基础模型的前提下，按需添加新能力。

### 8. 不足与局限
*   **分析器依赖**：系统的第一步依赖于 LLM 作为分析智能体，如果分析器无法准确识别失败原因，后续生成的环境将失效。
*   **路由上限**：目前的路由策略是基于任务描述的单次选择，对于需要在单次任务中交替使用多种极端专业能力的场景，可能需要更复杂的混合专家（MoE）路由机制。
*   **领域局限**：虽然在客服和工具使用上表现优异，但在代码生成或长程科学发现等更具发散性的领域，合成环境的自动构建难度会大幅增加。
